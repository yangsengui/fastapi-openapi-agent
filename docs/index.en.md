# OpenAPI should be more than API documentation

OpenAgent is an **OpenAPI-native API Agent**. It turns the OpenAPI definition of an existing FastAPI application into an Agent that can discuss the API, inspect exact operation contracts, and call real endpoints within an explicitly allowed security boundary.

```python
from openagent.fastapi import install_openapi_agent

install_openapi_agent(app)
```

This single line mounts an Agent page, JSON and SSE chat endpoints, an OpenAPI snapshot, and an embeddable sidebar. Open `/_agent/` to start using it.

## Why generate an Agent from OpenAPI?

Many API Agents require a second, handwritten set of tool names, parameters, and descriptions. That creates two sources of truth and quickly drifts when the API changes. OpenAgent consumes the OpenAPI document generated by the running application:

- `operationId` provides a stable tool identity;
- `summary`, `description`, and `tags` help the Agent select the right operation;
- parameters, request bodies, responses, and component schemas are loaded on demand;
- authentication settings determine which caller credentials may be forwarded;
- read-only and mutating operations follow different execution policies.

The quality ceiling of the Agent therefore starts with the quality of the OpenAPI definition. OpenAgent provides both the runtime integration and an engineering path where **the API contract defines the Agent's capabilities**.

## Core capabilities

| Capability | Description |
| --- | --- |
| One-line integration | Call `install_openapi_agent(app)` on an existing FastAPI app |
| Contract-driven | Build the operation catalog from `app.openapi()` without duplicate tool definitions |
| Progressive loading | Send compact metadata first and load the selected operation's schema on demand |
| Safe execution | Allow `GET`, `HEAD`, and `OPTIONS` by default; mutating calls require opt-in |
| In-process requests | Call the host API through ASGI without a public HTTP round trip |
| Provider-neutral models | Use OpenAI, DeepSeek, Anthropic, Gemini, and other providers through LiteLLM |
| Embeddable UI | Use a standalone page, iframe widget, or floating sidebar |
| Extensible SDK | Replace the model, policy layer, responder, or complete Agent backend |

## Shortest path to production

1. Improve the OpenAPI metadata exposed by your FastAPI endpoints.
2. Install `fastapi-openapi-agent[fastapi,llm]`.
3. Configure a model that supports tool calling.
4. Call `install_openapi_agent(app)`.
5. Validate in read-only mode before enabling any mutating operation.

## Project status

OpenAgent is currently alpha software. Public APIs may change before the first stable release. Pin the package version in production and carefully review credential forwarding, mutating-operation policies, and the data practices of your model provider.
